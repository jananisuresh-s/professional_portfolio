Smart Rooftop System

Embedded Systems  /  IoT  /  B.E. Final-Year Project

An ESP32-based automated rooftop that senses rain and temperature and closes itself accordingly, with Bluetooth-based manual and voice override. Built as a 4-member team project for our B.E. degree, submitted April 2026.

Contact me

Why we built this

Rooftops used for drying clothes, ventilation, or utility storage are fully exposed to the weather, and conventional setups depend entirely on someone noticing it's raining and going to close things manually. That delay causes real damage — soaked clothes, water ingress, heat buildup — and it's avoidable with fairly simple embedded sensing. We wanted to build a low-cost prototype that removes that manual dependency while still giving the user full override control when they want it.

How it works

The ESP32 microcontroller acts as the central control unit, continuously reading two sensors: a rain sensor and a DHT11 temperature/humidity sensor. When rain is detected, or temperature exceeds a 35°C threshold, the ESP32 signals a motor driver to close the rooftop via a DC motor. A limit switch detects the fully closed position and stops the motor, preventing mechanical overrun. Under normal conditions, the system reopens the rooftop automatically using timed motor control.

Alongside automatic operation, the system supports manual and voice control through the Arduino Bluetooth Control mobile app — a user can send commands like "Open 50%" or "Close fully" and the ESP32 interprets and executes them, overriding automatic behavior when needed. A 16×2 I²C LCD displays live temperature, humidity, rain status, and rooftop position.

System architecture

  • Sensing layer — rain sensor + DHT11 temperature/humidity sensor
  • Control layer — ESP32 compares sensor data against threshold conditions
  • Actuation layer — motor driver + DC motor, with limit switch for safe stopping
  • Communication layer — Bluetooth (built into ESP32) for manual and voice commands
  • Power layer — small solar panel + battery, demonstrating renewable power integration at prototype scale

Results

The prototype was tested under simulated rain, varying temperature, and manual Bluetooth commands. Rain detection triggered rooftop closure within roughly 1–2 seconds; the system correctly reopened under normal conditions and correctly deferred to manual override when a Bluetooth command was received. Combined automatic-and-manual testing showed no conflicts between the two control modes.

Limitations

  • DHT11 has moderate accuracy compared to industrial-grade sensors
  • Rain sensor can false-trigger on dust or ambient moisture without careful calibration
  • Prototype-scale motor and solar panel — not sized for a full residential rooftop
  • No cloud/IoT connectivity yet — control is local via Bluetooth only

Future scope

  • Replace DHT11 with a higher-accuracy environmental sensor for production use
  • Add IoT cloud connectivity for remote monitoring and control beyond Bluetooth range
  • Scale up motor and structural design for real rooftop dimensions and loads
  • Add a weatherproof enclosure for outdoor durability
