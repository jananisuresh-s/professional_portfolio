Brain Tumor Detection in MRI Scans

Computer Vision  /  Python

An end-to-end, rule-based pipeline that processes MRI scans and flags likely tumor regions — built independently to go deep on image processing rather than call a pretrained model and stop there.

View site View on GitHub

Why I built this

I wanted a project that forced me to understand image processing at the pixel level rather than treating computer vision as a black box. MRI tumor detection is a well-scoped, high-stakes-feeling problem: the input is messy real-world imaging data, and every stage of the pipeline (cleaning, isolating, segmenting, classifying) has a clear, explainable purpose. It's also a project I can defend line by line in an interview, because I built each stage myself rather than fine-tuning an existing model.

What it does

The pipeline takes a raw MRI scan and runs it through six stages to isolate and flag tumor regions:

  • Preprocessing — noise reduction and contrast normalization on the raw MRI image
  • Skull stripping — removing non-brain tissue so downstream stages only analyze relevant regions
  • Segmentation — isolating candidate regions of interest within the brain tissue
  • Feature extraction — pulling shape, intensity, and texture features from each candidate region
  • Rule-based classification — applying threshold and heuristic rules to flag regions as tumor or non-tumor
  • Result output — rendering the flagged regions back onto the scan through the Flask interface

Why rule-based, not deep learning

I chose a rule-based classification approach deliberately over a trained neural network. It meant I had to understand and hand-tune every decision boundary myself instead of relying on a model to learn one — which made the project slower to build but far easier to explain honestly. I can walk through exactly why a region gets flagged, which matters more to me right now than a marginally higher accuracy number I couldn't fully account for.

Future scope

  • Replace rule-based classification with a trained CNN classifier, benchmarked against the current rule-based baseline
  • Expand preprocessing to handle multiple MRI modalities (T1, T2, FLAIR) rather than a single scan type
  • Add a confidence score alongside each flagged region instead of a binary tumor/non-tumor output
  • Package the pipeline as a reusable Python library rather than a single Flask app
